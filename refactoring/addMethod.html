Add Method

Forces of Conflict

As it's such a fundamental pattern in object design that there are uncountably many situations that call for the addition of a method.

Perhaps a few lines of code are present in multiple other methods with no differences in implementaion; Maybe a class needs methods that get and set the values of instance variables; maybe a class simple needs to implement a new desired functionality. Consider the following code:

			public class aClass {
				string name;
				static string filename = "/testfile";

				public static void main(String[] args){
					this.name = "Michael";
					writeName();
					writeData("I can't think of example text");
				}
				public void writeData(string data){
					fw = new FileWriter(filename);
					BufferedWriter bw = new BufferedWriter(fw);
					bw.write(data);
					bw.close;
				}
				public void writeName(){
					fw = new FileWriter(filename);
					BufferedWriter bw = new BufferedWriter(fw);
					bw.write(this.name);
					bw.close;
				}
			}

It's apparent that the methods writeData and writeName are largely duplicated, with only minor differences between them.

Resolution

Adding the method write(string data) will provide a single place to consolidate the file stream operations.

			public class aClass {
				string name;
				static string filename = "/testfile";

				public static void main(String[] args){
					this.name = "Michael";
					write(this.name);
					write("I can't think of example text");
				}
				public void write(string data){
					fw = new FileWriter(filename);
					BufferedWriter bw = new BufferedWriter(fw);
					bw.write(data);
					bw.close;
				}
			}

Simpler and more dynamic, adding a method can greatly improve readability.
