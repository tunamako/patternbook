Create Empty Class

Forces of Conflict

A new class might need to be created for a number of reasons. Most often, it is a stepping stone in a chain of other design patterns. Perhaps an entirely new object needs to be distinguished, or a method shared between classes needs to be pulled into a single superclass.

Resolution

Very simple, but also powerful when in the context of other design patterns. Creating an empty class bears no inherent risk of affecting the behavior of code.
