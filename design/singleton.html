Singleton

Forces of Conflict

A class only needs to exist for one instance. In certain situations, it's more efficient to have a single instance of an object handle its class' behavior across the system, rather than have multiple instances active.

Resolution

Create a class constructor that only allows one instance of itself to be active at any given time. Ideally the constructor will be private so that it may only be called once, by the class itself at runtime. 

			public class Singleton {
				private static Singleton instance = new Singleton();

				private void Singleton() {

				}

				public static Singleton getInstance() {
					return instance;
				}
			}

This way, no outside object can construct a new Singleton object, but can access the already instantiated instance through getInstance().
